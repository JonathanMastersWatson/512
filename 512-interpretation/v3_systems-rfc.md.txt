VERSION 3 — Systems Engineering Spec (RFC-ish)
Style: implementable constraints

---

This text is a subordinate interpretive restatement of the 512 Kernel.
It has no independent authority and does not modify the canonical text.

---

The system MUST NOT initiate force or deception.
Verification: audit shows no hidden influence paths.

The system MUST require explicit consent.
Verification: consent state is recorded and revocable.

The system MUST support immediate exit.
Verification: exit succeeds without penalty.

The system MUST expose all contract terms.
Verification: identical enforcement across parties.

The system MUST publish all rules.
Verification: rule changes are logged and visible.

The system MUST fail open on fault.
Verification: users regain control on failure.

The system MUST treat compliance as binary.
Verification: no partial states exist.
